# Instrucciones para Codex

## Propósito

Este es un proyecto educativo desarrollado por un estudiante trainee. El objetivo es construir una aplicación funcional comprendiendo las decisiones tomadas.

## Reglas

1. Trabajar únicamente dentro de este repositorio.
2. No modificar archivos externos.
3. No agregar funcionalidades no solicitadas.
4. Trabajar de a una tarea por vez.
5. Inspeccionar antes de modificar.
6. Informar qué archivos serán modificados.
7. Mantener el proyecto ejecutable.
8. No instalar dependencias adicionales sin autorización.
9. No hacer refactorizaciones prematuras.
10. No realizar commits sin autorización.
11. No ejecutar `push` sin autorización.
12. No cambiar las tecnologías definidas para el MVP.

## Tecnologías permitidas

* React;
* Vite;
* JavaScript;
* CSS;
* localStorage;
* Git;
* GitHub.

## Tecnologías prohibidas durante el MVP

* TypeScript;
* Next.js;
* Tailwind;
* Bootstrap;
* Redux;
* React Router;
* backend;
* autenticación;
* bases de datos remotas;
* inteligencia artificial;
* librerías grandes de componentes.

## Arquitectura inicial

* aplicación React de una sola pantalla;
* una única fuente de verdad para las postulaciones;
* componentes con responsabilidades claras;
* datos derivados para estadísticas y filtros;
* acceso a `localStorage` aislado detrás de un servicio;
* CSS propio;
* sin sobreingeniería.

## Orden de desarrollo

1. estructura visual;
2. modelo y constantes;
3. formulario;
4. creación y listado;
5. edición;
6. eliminación;
7. validaciones;
8. persistencia;
9. búsqueda;
10. filtro;
11. estadísticas;
12. seguimientos;
13. responsive;
14. pruebas;
15. documentación;
16. publicación.

## Método de explicación

Ante cada concepto nuevo, explicar:

1. el problema;
2. la solución;
3. la alternativa;
4. el aprendizaje;
5. la defensa en entrevista.

## Finalización de tareas

Después de cada tarea:

1. ejecutar las verificaciones necesarias;
2. resumir los cambios;
3. mostrar `git diff --stat`;
4. mostrar `git status`;
5. detenerse y esperar autorización.
